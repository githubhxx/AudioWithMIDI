# Scheme3 评估结果详细分析

## 当前结果摘要

| 指标 | 数值 | 含义 |
|------|------|------|
| mel_l1 | 5.74 | Mel 频谱 L1 距离（帧内平均绝对误差） |
| mel_l2 | 38.83 | Mel 频谱 L2 距离（帧内均方误差） |
| mel_sc | **0.99987** | 谱收敛度 |
| mel_log | 5.04 | 对数 Mel 距离 |
| energy_corr | **0.0035** | 能量包络相关系数 |
| fd_mel | 2829.16 | Mel 帧特征空间上的 Fréchet 距离 |
| kl_mel | **4107181.37** | Mel 帧特征空间上的 KL(参考‖生成) |

---

## 一、各指标含义与“正常”范围

- **mel_l1 / mel_l2 / mel_log**：生成 Mel 与参考 Mel 的逐点距离，**越小越好**，0 表示完全一致。参考为 log-Mel，数值多在约 [-10, 5] 量级，故 L1≈5.74 表示平均每 bin 误差较大。
- **mel_sc（谱收敛度）**：公式为 `||M_ref - M_gen||_F / ||M_ref||_F`，**越小越好**，0 表示生成与参考一致。
- **energy_corr**：能量包络的相关系数，范围 **[-1, 1]**，**越接近 1 越好**，表示动态/起伏一致。
- **fd_mel**：Fréchet 距离，**越小越好**，表示两分布（参考 vs 生成）在 Mel 帧特征空间中的分布距离。
- **kl_mel**：KL(参考分布 ‖ 生成分布)，**越小越好**，表示用高斯近似后“参考相对生成的相对熵”。

---

## 二、异常问题分析

### 1. mel_sc ≈ 0.99987（严重异常）

- **含义**：分子 `||M_ref - M_gen||` 几乎等于分母 `||M_ref||`，即“生成与参考的差异”几乎和“参考本身的范数”一样大。
- **可能原因**：
  - 生成 Mel 与参考 Mel 在数值上相差极大（例如尺度/量纲不一致，或生成整体偏小/偏大）。
  - 生成与参考在**时间上严重不对齐**：评估时用 `_ensure_same_length` 只按**最短长度**截断后逐帧对比，若生成长度与参考长度差异大，相当于用“参考的前一段”去比“生成的整段”或反过来，语义不对齐会导致差异人为放大。
- **结论**：当前生成与参考在 Mel 空间上**极不一致**，和“理想情况（mel_sc 接近 0）”相差很远，属于明显异常。

### 2. energy_corr ≈ 0.0035（严重异常）

- **含义**：能量包络几乎无线性相关（接近 0），即生成的响度/动态变化与参考几乎不相关。
- **可能原因**：
  - **定义不一致**：评估里 `energy_gen = sum(|mel_gen|, axis=0)`，而 `energy_ref` 来自 STFT 的 `norm(magnitudes, dim=1)`，二者物理含义和尺度不同，即使对齐也可能相关度低。
  - **时间不对齐**：同上，若生成与参考的 T 差异大，截断后比较的已不是“同一段音乐”的起伏。
  - **模型未学好动态**：条件生成未抓住参考的强弱与节奏变化。
- **结论**：从指标上看，**能量/动态一致性极差**，属于异常；若只改评估脚本，也需统一能量定义并注意对齐方式。

### 3. kl_mel ≈ 4.1e6（严重异常）

- **含义**：在 Mel 帧特征 (T, 80) 上，用高斯近似参考分布 P、生成分布 Q 后，KL(P‖Q) 达到百万级。
- **为何会爆炸**：  
  KL(P‖Q) 公式中含：
  - `log(det(Sigma_Q) / det(Sigma_P))`  
  - 若 **生成分布的协方差矩阵 Sigma_Q 几乎奇异**（特征值有接近 0 的），则 `det(Sigma_Q)` 接近 0，该项会趋向负无穷，整体 KL 会极大甚至数值爆炸。
- **典型成因**：**生成分布坍缩（distribution collapse）**：模型生成的多段/多帧 Mel 非常相似，帧间方差很小，导致用帧向量估计的协方差矩阵秩不足或条件数很差，从而：
  - 生成分布被拟合成一个“很尖”的高斯；
  - 参考分布相对更分散；
  - KL(参考‖生成) 就会极大。
- **结论**：**kl_mel 数量级异常**，强烈提示“生成 Mel 帧的多样性严重不足”，与 fd_mel 偏大一起看，说明生成分布与参考分布差异大且生成侧分布过于集中，属于需要优先关注的异常。

### 4. fd_mel = 2829（偏大，需结合分布看）

- **含义**：在 80 维 Mel 帧特征上，参考与生成两分布之间的 Fréchet 距离。
- **为何偏大**：与 kl_mel 一致，若生成分布坍缩或均值/方差与参考分布差异大，FD 会变大。2829 在 80 维、方差量级未归一化时可以是“两分布相距很远”的量级。
- **结论**：与 kl_mel、mel_sc 一起，**支持“生成分布与参考分布差异大、且生成多样性不足”** 的判断。

### 5. mel_l1 / mel_l2 / mel_log（偏大但非“爆炸”）

- 数值上明显大于“生成与参考接近”时的水平，说明**逐点 Mel 相似度差**，与 mel_sc、fd_mel、kl_mel 一致。
- 可能同时受**尺度不一致、时间不对齐、生成质量不足**影响，但不像 kl_mel 那样直接暴露数值/分布问题。

---

## 三、根因归纳

1. **生成分布坍缩**  
   生成 Mel 帧过于相似 → 协方差近似奇异 → **kl_mel 爆炸**，fd_mel 偏大。

2. **时间对齐可能不当**  
   生成长度由 MIDI 条件与压缩因子决定，参考长度由 WAV 决定；当前仅按最短 T 截断后逐帧比较，若两者长度或起止不对齐，会人为放大 mel_* 距离并拉低 energy_corr。

3. **能量定义不一致**  
   energy_gen（Mel 绝对值求和）与 energy_ref（STFT 幅度范数）不一致，会压低 energy_corr，难以单独解读“模型是否学好动态”。

4. **生成质量/训练不足**  
   若扩散阶段或 VAE 解码尚未学好，会直接导致生成 Mel 与参考在尺度、形状、分布上都不接近，表现为所有 Mel 相关指标变差。

---

## 四、建议（优先顺序）

1. **排查生成多样性**  
   - 可视化多段生成的 Mel（或听感），看是否过于平坦、重复。  
   - 若存在坍缩：检查扩散训练（学习率、步数、CFG）、潜在空间维度与数据多样性。

2. **稳定 KL 计算（短期）**  
   - 在 `compute_feature_stats` / 协方差估计中对角线加更大正则（如 `eps=1e-4` 或更大），或对特征做 PCA 降维后再算 KL，避免奇异协方差导致 kl_mel 爆炸；同时报告“是否曾触发数值保护”便于判断坍缩。

3. **评估对齐与长度**  
   - 统计并打印：每对 (midi, ref) 的生成帧数 T_gen 与参考帧数 T_ref，以及截断后用于比较的长度。  
   - 若 T_gen 与 T_ref 经常相差很大，考虑按时间对齐（例如按 MIDI 时间或按能量对齐）再算 mel_* 与 energy_corr，或只对“长度接近”的配对做汇总。

4. **统一能量定义**  
   - 若希望 energy_corr 有可比性：用同一套定义计算 ref 与 gen 的能量（例如都用 STFT 幅度范数，或都用同一 Mel 的 sum(abs(·))），并尽量在时间对齐后的帧上计算。

5. **复测与基线**  
   - 修复或缓解上述问题后，用同一批 (midi, ref) 重新跑评估，观察 mel_sc、energy_corr、kl_mel、fd_mel 是否进入合理范围；若有基线模型或 Tango 官方指标，可对比同一数据上的数量级。

---

## 五、总结表

| 指标 | 是否异常 | 主要可能原因 |
|------|----------|----------------|
| mel_sc | 是（≈1） | 生成与参考差异极大；时间/尺度不对齐 |
| energy_corr | 是（≈0） | 能量定义不一致；时间不对齐；动态未学好 |
| kl_mel | 是（百万级） | 生成分布坍缩，协方差近似奇异 |
| fd_mel | 偏大 | 生成与参考分布相距远；与坍缩一致 |
| mel_l1/l2/log | 偏大 | 生成质量/对齐/尺度问题 |

整体来看，**最突出的异常是 kl_mel 爆炸和 mel_sc≈1、energy_corr≈0**，建议优先从“生成分布多样性”和“评估对齐/能量定义”两方面排查与改进。
